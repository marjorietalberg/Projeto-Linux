Sobre o

Projeto

O projeto envolve criar uma infraestrutura na AWS com VPC, instância EC2, servidor Nginx e uma página HTML. A VPC deve ter duas sub-redes públicas e privadas, e a instância EC2 deve usar Ubuntu ou Amazon Linux 2023. O servidor Nginx será configurado para exibir uma página HTML simples. Além disso, será criado um script de monitoramento que verifica a disponibilidade do site a c ada 1 minuto e envia notificações via Discord, Telegram ou Slack se o serviço estiver fora do ar. O script também armazenará logs em /var/log/meu_script.log. Após testar, o processo deve ser documentado no GitHub. Como desafios bônus, a configuração será automatizada via User Data e um arquivo CloudFormation será criado para provisionar a infraestrutura automaticamente.
Etapas do Projeto
Criar uma VPC com sub-redes públicas e privadas, lançar uma instância EC2 com Ubuntu.
Desenvolver um script que verifica a disponibilidade do site a cada 1 minuto, envia notificações via Discord,
Testar a implementação, documentar o processo no GitHub e automatizar a configuração via User Data
Notas

Notas sobre o Projeto: Este projeto consiste em uma série de etapas detalhadas para criar e configurar uma infraestrutura na AWS, com foco em ambientes de nuvem e automação de monitoramento. Abaixo estão os principais pontos de cada fase do desenvolvimento:

Etapa 1: Configuração do Ambiente A primeira fase do projeto envolve a criação de uma VPC (Virtual Private Cloud) na AWS, configurando duas sub-redes públicas e duas privadas para garantir a segurança e a segmentação adequada do tráfego. Além disso, uma instância EC2 é lançada dentro da sub-rede pública utilizando o sistema operacional Ubuntu ou Amazon Linux 2023, criando a base para o ambiente de servidor.

Etapa 2: Configuração do Servidor Web Nessa fase, um servidor Nginx é instalado e configurado na instância EC2. Em seguida, é criada uma página HTML simples, que será exibida pelo Nginx, garantindo que a instância EC2 esteja funcionando corretamente e disponibilizando uma interface básica ao usuário.

Etapa 3: Script de Monitoramento + Webhook Aqui, é desenvolvido um script de monitoramento que verifica a cada minuto se o site está ativo. Caso o serviço esteja fora do ar, o script envia uma notificação de falha para canais como Discord, Telegram ou Slack. Além disso, o script armazena logs de execução em um local específico, como /var/log/meu_script.log, para facilitar o acompanhamento e resolução de problemas.

Etapa 4: Testes e Documentação Após a implementação, é importante realizar testes para garantir que todos os componentes estão funcionando corretamente. A documentação do processo de instalação e configuração é feita no GitHub, com atenção especial para a segurança, a fim de não comprometer dados sensíveis.
